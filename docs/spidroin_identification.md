# 蛛丝蛋白鉴定流程

基于jbrowse2 基因组浏览器的蛛丝蛋白鉴定流程，主要包括以下步骤：

1. 目前已有的数据源有：

   - 134个物种的基因组和注释数据，来源于NCBI和Ensembl数据库。
   - 重新对134个物种的新注释
   - nhmmer search结果，包含了蛛丝蛋白的NTD和CTD的搜索结果。
   - miniprot search结果，包含了蛛丝蛋白的NTD和CTD的搜索结果。
   - 134个物种的二代测序数据，来源于SRA数据库和自己测的数据（bigwig和bam文件）。
   - 部分（10个）物种的mRNA三代测序数据，来源于自己测的数据（bigwig和bam文件）。

2. 首先打开一个物种的基因组浏览器，选择一个物种（例如：**Argiope bruennichi**），进入该物种的基因组浏览器界面。

3. 在基因组浏览器中，选择一个染色体（例如：**chr1**），按照一下顺序依次打开track：

    - 基因组序列（genome sequence）
    - nhmmer search结果，选择NTD和CTD的搜索结果
    - miniprot search结果，选择NTD和CTD的搜索结果
    - 134个物种的二代测序数据，选择bigwig
    - 部分（10个）物种的mRNA三代测序数据，选择bigwig
    - 基因组注释文件，新版和旧版。

4. 证据源优先级：nhmmer search结果 > miniprot search结果  > 基因组注释文件 > 二代测序数据 = mRNA三代测序数据

5. 案例：

    - 一般案例通过nhmmer search结果就能得到：在nhmmer search结果中，NTD和CTD的搜索结果很整齐，NTD的前三个碱基都对应着基因组上的起始密码子，CTD的最后三个碱基都对应着基因组终止密码子前三个碱基，且NTD和CTD的搜索结果之间没有其他的搜索结果，长度大于 1kb，小于150kb，这种情况就可以认为这个基因是一个蛛丝蛋白基因。
    - 如果没有nhmmer search结果，可以通过miniprot search结果来鉴定蛛丝蛋白基因，miniprot search结果的优先级低于nhmmer search结果，如果miniprot search结果中NTD和CTD的搜索结果很整齐，NTD的前三个碱基都对应着基因组上的起始密码子，CTD的最后三个碱基都对应着基因组上的终止密码子，因为miniprot的比对质量差，所以中间有其他的比对结果也没问题，只要两端之间的长度大于 1kb，小于150kb，这种情况就可以认为这个基因是一个蛛丝蛋白基因。
    - 如果nhmmer search结果和miniprot search结果的两端没有和起始密码子和终止密码子对齐，或者只有一端和基因组注释文件对齐，那么可以查看对应的基因组注释文件，如果nhmmer search结果和miniprot search结果都在基因组注释区域内，并且这个基因的长度大于 1kb，小于150kb，这种情况就可以认为这个基因是一个蛛丝蛋白基因。
    - 如果以上方法都无法确定，可以结合多种证据源进行综合判断，最终要确保这个基因的长度大于 1kb，小于150kb，两端必须对应着起始密码子和终止密码子，且有足够的证据支持它是一个蛛丝蛋白基因。

6. 蛛丝蛋白类型的判断：

    - nhmmer search结果看E-value，E-value小于1e-10的结果可以认为高置信度的，如果多个类型重叠，优先考虑E-value较小的类型。
    - miniprot search结果看比对同源性 Positive 字段（0-1），优先考虑同源性较高的类型。
    - 如果两端的搜索结果类型不一致，结合附近的搜索结果类型进行综合判断，一般离得比较近的都是同一类型的蛛丝蛋白。
    - 如果两端的搜索结果类型不一致，且附近没有其他的搜索结果，或者附近的搜索结果类型也不一致，那么这种需要人工凭借经验进行综合判断。
